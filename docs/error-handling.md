## 1. エラー分類

エラーは以下のカテゴリに分類します。ユーザー通知文は日本語短文で統一し、内部コードは実装・ログ管理に使用します。

| カテゴリ | 概要 | 対応者 |
|---------|------|-------|
| INPUT_ERROR | 画像破損・形式不正など、入力ファイルの問題 | ユーザー |
| OCR_ERROR | OCR 失敗・文字読取不足など、抽出処理の問題 | ユーザー / 再実行 |
| VALIDATION_ERROR | 必須項目不足・金額不整合など、検証で検出された問題 | ユーザー |
| CONFIG_ERROR | フォルダ ID 不正・設定不足など、設定の問題 | 管理者 |
| SYSTEM_ERROR | API 障害・Drive 操作失敗など、システムの一時的な問題 | 管理者 / 再実行 |

## 2. エラー通知文一覧

### 入力系（INPUT_ERROR）

| 内部コード | 通知文 | 想定原因 | 再処理可否 |
|-----------|--------|---------|----------|
| `IMAGE_UNREADABLE` | 画像不鮮明、要再アップロード | ピンぼけ・低照度・反射 | 差替後に再投入 |
| `IMAGE_CORRUPT` | 画像破損、要差替 | ファイル破損・転送エラー | 差替後に再投入 |
| `FORMAT_UNSUPPORTED` | 形式不正、要差替 | 非対応ファイル形式 | 差替後に再投入 |
| `FILE_NOT_FOUND` | ファイル取得失敗、管理者確認 | Drive アクセス権限・削除 | 管理者確認後 |

### OCR / 抽出系（OCR_ERROR）

| 内部コード | 通知文 | 想定原因 | 再処理可否 |
|-----------|--------|---------|----------|
| `OCR_FAILED` | OCR失敗、要再実行 | API 一時障害・タイムアウト | 再投入可 |
| `TEXT_INSUFFICIENT` | 文字読取不足、要確認 | 画像品質低下・手書き文字 | 差替後に再投入 |
| `DATE_NOT_FOUND` | 日付読取不可、要確認 | 日付が写っていない・読取不可 | 差替後に再投入 |
| `AMOUNT_NOT_FOUND` | 金額読取不可、要確認 | 金額が写っていない・読取不可 | 差替後に再投入 |
| `STORE_NOT_FOUND` | 店名読取不可、要確認 | 店名部分の画像品質低下 | 差替後に再投入 |

### 検証系（VALIDATION_ERROR）

| 内部コード | 通知文 | 想定原因 | 再処理可否 |
|-----------|--------|---------|----------|
| `REQUIRED_FIELD_MISSING` | 必須項目不足、要確認 | 証憑に項目の記載なし | 差替後に再投入 |
| `AMOUNT_INCONSISTENT` | 金額不整合、要確認 | 合計と内訳の不一致 | 差替後に再投入 |
| `DATE_INVALID` | 日付不正、要確認 | 日付の形式・値が不正 | 差替後に再投入 |
| `DUPLICATE_SUSPECTED` | 重複疑い、要確認 | 同一 Drive ファイル ID が台帳に存在 | 手動確認後 |

### 設定系（CONFIG_ERROR）

| 内部コード | 通知文 | 想定原因 | 再処理可否 |
|-----------|--------|---------|----------|
| `CONFIG_INVALID` | 設定不備、管理者確認 | フォルダ ID・シート ID・API キーの設定不足 | 管理者修正後 |
| `FOLDER_NOT_FOUND` | フォルダ取得失敗、管理者確認 | フォルダ ID 不正・削除 | 管理者確認後 |

### システム系（SYSTEM_ERROR）

| 内部コード | 通知文 | 想定原因 | 再処理可否 |
|-----------|--------|---------|----------|
| `SHEET_WRITE_FAILED` | 登録失敗、管理者確認 | Sheets への書込失敗 | 再実行可 |
| `FILE_MOVE_FAILED` | フォルダ移動失敗、管理者確認 | Drive 移動操作失敗 | 再実行可 |
| `API_UNAVAILABLE` | 一時エラー、要再実行 | Gemini API 一時障害 | 再実行可 |
| `QUOTA_EXCEEDED` | API制限超過、管理者確認 | Gemini API クォータ超過 | 管理者確認後 |

## 3. ユーザー対応フロー

### `04_エラー` フォルダのファイルを確認したとき

1. エラーログシートで対象ファイルの通知文を確認する
2. 通知文に応じて以下を実施する

| 通知文の末尾 | ユーザーの対応 |
|------------|-------------|
| 要再アップロード / 要差替 | 鮮明な画像に差し替えて `01_取込` に再アップロード |
| 要確認 | 元の証憑を確認し、内容を手動でスプレッドシートに入力 |
| 要再実行 | `01_取込` にファイルを戻し、次のトリガー実行を待つ |
| 管理者確認 | 担当管理者に連絡する |

## 4. 管理者対応フロー

### CONFIG_ERROR が発生した場合

1. GAS スクリプトの `CONFIG` オブジェクトを確認する
2. フォルダ ID・シート ID が正しいか検証する
3. `GEMINI_API_KEY` が `CONFIG` オブジェクトに正しく設定されているか確認する（プレースホルダーのままになっていないか確認する）

### SYSTEM_ERROR が継続する場合

1. GAS の実行ログを確認する
2. Google Workspace 側の障害情報を確認する
3. Gemini API のクォータ使用状況を確認する

## 5. エラーログ記録仕様

エラーが発生した場合、スプレッドシートのエラーログシートに以下を記録します。

| 項目 | 内容 |
|------|------|
| 発生日時 | GAS が処理した日時（YYYY-MM-DD HH:mm:ss） |
| ファイル名 | エラーが発生したファイルの名前 |
| Drive ファイル ID | Drive 上のファイル ID（一意識別子） |
| エラーカテゴリ | INPUT_ERROR / OCR_ERROR / VALIDATION_ERROR / CONFIG_ERROR / SYSTEM_ERROR |
| 内部コード | エラーの詳細コード（例：`DATE_NOT_FOUND`） |
| 通知文 | ユーザー向け日本語短文 |
| 再処理可否 | 可 / 要管理者確認 |
| 対応ステータス | 未対応 / 対応中 / 解決済 |

## 6. 再処理手順

### 通常の再処理

1. `04_エラー` フォルダから対象ファイルを `01_取込` フォルダに移動する
2. GAS の次のトリガー実行で自動処理される

### 重複チェックに引っかかった場合

1. エラーログシートで対象ファイルの Drive ファイル ID を確認する
2. 台帳シートで同一 ID が既に登録済みかを確認する
3. 登録済みであれば重複のため処理不要、未登録であれば管理者に報告する
