## 1. 全体アーキテクチャ

### 1.1 コンポーネント構成

```
┌─────────────────────────────────────────────────────────┐
│                  Google Workspace                        │
│                                                          │
│  ┌──────────────┐    ┌─────────────────────────────┐    │
│  │ Google Drive  │    │  Google Apps Script (GAS)   │    │
│  │               │    │                             │    │
│  │ 01_取込        │◄──┤  main() トリガー定期実行     │    │
│  │ 02_処理済み    │    │  ↓                          │    │
│  │               │    │  DriveService（取得・移動）  │    │
│  └──────────────┘    │  ↓                          │    │
│                       │  GeminiService（OCR・抽出）  │◄──┼──► Gemini API
│  ┌──────────────┐    │  ↓                          │    │    (gemini-2.5-flash-lite)
│  │ Google Sheets │    │  ParseService（パース）      │    │
│  │               │◄──┤  ↓                          │    │
│  │ 経費証憑台帳  │    │  ValidationService（検証）  │    │
│  │               │    │  ↓                          │    │
│  └──────────────┘    │  SheetsService（追記）       │    │
│                       │  ↓                          │    │
│                       │  FileService（移動・命名）   │    │
│                       └─────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

### 1.2 処理フロー

```
[開始] GAS タイムトリガー発火（例: 10分おき）
  │
  ▼
[1] 取込フォルダ内の未処理ファイル一覧を取得
  │  ・JPEG / PNG / HEIC / WebP を対象
  │  ・処理上限件数に達したら後続ファイルは次回実行へ
  │
  ▼
[2] ファイルごとにループ処理（try-catch で 1 件失敗しても続行）
  │
  ├─[3] 精算対象月を決定（CONFIG 固定値またはフォルダ名から取得）
  │
  ├─[4] Gemini API に画像を送信し、OCR + 構造化抽出を実行
  │      ・JSON スキーマを指定してレスポンスを受け取る
  │      ・タイムアウト・API エラー時は要確認として記録
  │
  ├─[5] レスポンスを内部データ構造にパース
  │      ・型変換（数値・日付）
  │      ・null / 空文字の正規化
  │
  ├─[6] バリデーション
  │      ・必須項目チェック
  │      ・金額整合性チェック（合計 ≒ 10%内訳 + 8%内訳 + 非課税 + 不明）
  │      ・要確認フラグ・要確認理由の付与
  │
  ├─[7] スプレッドシートに 1 行追記
  │      ・処理日時・精算対象月・画像 URL・申請者を GAS 側で付与
  │
  └─[8] ファイルをリネームして処理済みフォルダへ移動

  ▼
[9] ログ記録（処理結果・エラーをログシートまたは Logger に記録）

[終了]
```

## 2. モジュール設計

GAS の `Code.gs`（または複数ファイル）を以下の責務ごとに分けて実装する。小規模であれば単一ファイルでの実装も可だが、関数単位での責務分離を徹底する。

### 2.1 設定管理（`CONFIG`）

```javascript
const CONFIG = {
  INTAKE_FOLDER_ID: 'ここに取込フォルダIDを設定',
  PROCESSED_FOLDER_ID: 'ここに処理済みフォルダIDを設定',
  SPREADSHEET_ID: 'ここにスプレッドシートIDを設定',
  SHEET_NAME: '経費証憑台帳',
  LOG_SHEET_NAME: '処理ログ',
  EXPENSE_MONTH_MODE: 'fixed',   // 'fixed' または 'folder'
  EXPENSE_MONTH_FIXED: '2026-03', // EXPENSE_MONTH_MODE = 'fixed' の場合のみ使用
  APPLICANT_NAME: 'ここに申請者名を設定',
  MAX_FILES_PER_RUN: 10,
  GEMINI_MODEL: 'gemini-2.5-flash-lite'
};
```

- `GEMINI_API_KEY` はスクリプトプロパティで管理し、`CONFIG` には含めない
- 実際の ID・キーはコミットしない（プレースホルダー文字列を置く）

### 2.2 Drive 画像取得（`DriveService`）

| 関数 | 責務 |
|------|------|
| `getUnprocessedFiles(folderId)` | 取込フォルダから未処理の画像ファイル一覧を取得する |
| `getExpenseMonth(file, mode)` | 精算対象月を決定する（CONFIG 固定値またはフォルダ名から取得） |
| `getFileAsBase64(file)` | ファイルを Base64 エンコードして返す（Gemini API 送信用） |

### 2.3 Gemini API 呼び出し（`GeminiService`）

| 関数 | 責務 |
|------|------|
| `callGeminiOcr(base64Image, mimeType)` | Gemini API に画像を送信し、構造化 JSON レスポンスを受け取る |
| `buildPrompt()` | OCR + 構造化抽出のためのプロンプトを生成する |

- API キーは `PropertiesService.getScriptProperties().getProperty('GEMINI_API_KEY')` で取得
- レスポンスは JSON 文字列として返ってくることを前提とし、パースエラーは上位でハンドリングする

### 2.4 OCR 結果パース（`ParseService`）

| 関数 | 責務 |
|------|------|
| `parseGeminiResponse(jsonString)` | Gemini のレスポンス JSON を内部データオブジェクトにパースする |
| `normalizeDate(value)` | 日付文字列を `YYYY-MM-DD` に正規化する |
| `normalizeAmount(value)` | 金額文字列を数値に変換する（カンマ除去・全角数字変換等） |

### 2.5 書類種別判定

書類種別の判定は Gemini API 側で行い、`document_type` フィールドとして返却する。GAS 側では返却値の検証のみ行う。

有効な値：`レシート` / `領収書` / `簡易領収書` / `利用明細` / `不明`

### 2.6 バリデーション（`ValidationService`）

| 関数 | 責務 |
|------|------|
| `validate(data)` | 抽出データに対して要確認判定ルールを適用し、`needs_review` と `review_reason` を付与して返す |

詳細な判定ルールは [5. 要確認判定ルール](#5-要確認判定ルール) を参照。

### 2.7 Sheets 追記（`SheetsService`）

| 関数 | 責務 |
|------|------|
| `appendRow(spreadsheetId, sheetName, rowData)` | スプレッドシートの末尾に 1 行追記する |
| `buildRowArray(data, meta)` | 内部データオブジェクトから列定義順の配列を生成する |

`meta` には GAS 側で付与する処理日時・精算対象月・申請者・画像 URL を含む。

### 2.8 Drive 移動・リネーム（`FileService`）

| 関数 | 責務 |
|------|------|
| `moveToProcessed(file, folderId, newName)` | ファイルをリネームして指定フォルダへ移動する |
| `buildFileName(status, processedDate, originalFileName)` | [6. ファイル命名規則](#6-ファイル命名規則) に従いファイル名を生成する（`status` は `'処理済'` / `'要確認'` / `'エラー'`） |

### 2.9 ログ・エラーハンドリング（`LogService`）

| 関数 | 責務 |
|------|------|
| `logInfo(message)` | 処理情報をログシートまたは `Logger.log` に記録する |
| `logError(fileName, error)` | エラー内容をログシートに記録する |

- ファイル単位で try-catch を設け、1 件の失敗が後続処理に影響しないようにする
- GAS 標準のエラーメール通知（トリガー設定でのエラー通知）を活用する

## 3. データ設計

### 3.1 スプレッドシート列定義

シート名：`経費証憑台帳`（`CONFIG.SHEET_NAME` で変更可能）

| 列 | フィールド名 | 型 | 必須 | 形式・説明 |
|----|------------|-----|------|-----------|
| A | 処理日時 | 日時 | ○ | `YYYY-MM-DD HH:mm:ss`。GAS が付与 |
| B | 精算対象月 | テキスト | ○ | `YYYY-MM`（例: `2026-03`）。GAS が CONFIG またはフォルダ名から付与 |
| C | 書類種別 | テキスト | — | `レシート` / `領収書` / `簡易領収書` / `利用明細` / `不明` |
| D | 利用日 | テキスト | — | `YYYY-MM-DD`。Gemini が抽出 |
| E | 発行日 | テキスト | — | `YYYY-MM-DD`。Gemini が抽出 |
| F | 支払先名 | テキスト | — | Gemini が抽出 |
| G | 宛名 | テキスト | — | Gemini が抽出。「上様」を含む |
| H | 摘要 / 但し書き | テキスト | — | Gemini が抽出 |
| I | 合計金額（税込） | 数値 | — | 円。Gemini が抽出 |
| J | 10%対象額 | 数値 | — | 円。Gemini が抽出 |
| K | 10%税額 | 数値 | — | 円。Gemini が抽出 |
| L | 8%対象額 | 数値 | — | 円。Gemini が抽出（軽減税率） |
| M | 8%税額 | 数値 | — | 円。Gemini が抽出（軽減税率） |
| N | 非課税額 | 数値 | — | 円。Gemini が抽出 |
| O | 税率不明額 | 数値 | — | 円。Gemini が抽出 |
| P | 主税率区分 | テキスト | — | `10%` / `8%` / `混在` / `非課税` / `不明` |
| Q | 支払方法 | テキスト | — | `現金` / `クレジットカード` / `電子マネー` / `QR決済` / `不明` 等 |
| R | 書類番号 | テキスト | — | 領収書番号・伝票番号等 |
| S | 適格請求書発行事業者登録番号 | テキスト | — | `T` + 13 桁数字 |
| T | 住所 | テキスト | — | 発行者の住所 |
| U | 電話番号 | テキスト | — | 発行者の電話番号 |
| V | 申請者 | テキスト | — | CONFIG から付与 |
| W | 画像 URL | URL | ○ | Google Drive ファイル URL。GAS が付与 |
| X | OCR 抽出テキスト | テキスト | — | Gemini が認識した全テキスト |
| Y | 要確認フラグ | テキスト | ○ | `TRUE` / `FALSE` |
| Z | 要確認理由 | テキスト | — | 要確認と判断した理由（改行区切り） |
| AA | 備考 | テキスト | — | 申請者が手動で入力するメモ欄 |

> **精算対象月（B列）の型・形式**：テキスト型、`YYYY-MM` 形式（例: `2026-03`）。スプレッドシート上で月次フィルタ・ピボット集計に使用できる形式とする。

> **税率関連項目（J〜P列）の定義**：数値型列（J〜O）は円単位の整数値を格納する。税率区分が判別できない場合は空欄とし、`税率不明額`（O列）に当該金額を格納する。`主税率区分`（P列）は証憑全体で最も支配的な税率区分を示す概要フィールドである。

### 3.2 ログシート列定義

シート名：`処理ログ`（`CONFIG.LOG_SHEET_NAME` で変更可能）

| 列 | フィールド名 | 説明 |
|----|------------|------|
| A | 処理日時 | `YYYY-MM-DD HH:mm:ss` |
| B | ファイル名 | 処理対象のファイル名 |
| C | 処理結果 | `成功` / `要確認` / `エラー` |
| D | メッセージ | エラー内容・要確認理由等 |

## 4. Gemini 抽出仕様

### 4.1 基本方針

- **OCR と構造化 JSON 抽出を 1 回の API コールで実行する**
- 証憑の記載内容から読み取れる情報のみ抽出し、**推測・補完は行わない**
- 記載がない項目・読み取れない項目は `null` または空文字で返す
- 書類種別・要確認フラグ・要確認理由も JSON に含めて返す

### 4.2 プロンプト設計方針

Gemini に送信するプロンプトは以下の要素を含む。

1. 「経費証憑の画像から情報を抽出する」という役割の明示
2. 返却する JSON スキーマの指定
3. 不明項目は `null` で返すよう指示
4. 推測・補完をしないよう指示
5. `needs_review` と `review_reason` を含めるよう指示

### 4.3 書類種別の候補

Gemini が `document_type` フィールドで返す有効な値：

| 値 | 説明 |
|----|------|
| `レシート` | 販売・飲食等のレシート |
| `領収書` | 宛名・但し書きが記載された正式領収書 |
| `簡易領収書` | 宛名を省略した簡略領収書 |
| `利用明細` | クレジットカード明細・交通系 IC 明細等 |
| `不明` | 上記に分類できない、または画像から判断できない |

### 4.4 JSON スキーマ例

```json
{
  "document_type": "レシート",
  "usage_date": "2026-03-18",
  "issue_date": null,
  "vendor_name": "○○スーパー 渋谷店",
  "addressee": null,
  "description": "食料品・日用品",
  "total_amount": 1980,
  "tax_10_base": 500,
  "tax_10_amount": 50,
  "tax_8_base": 1300,
  "tax_8_amount": 104,
  "tax_exempt_amount": null,
  "tax_unknown_amount": null,
  "primary_tax_rate": "混在",
  "payment_method": "クレジットカード",
  "document_number": null,
  "invoice_registration_number": "T1234567890123",
  "address": "東京都渋谷区○○1-2-3",
  "phone": "03-0000-0000",
  "ocr_text": "○○スーパー 渋谷店\n2026/03/18 13:45\n...",
  "needs_review": false,
  "review_reason": null
}
```

### 4.5 レスポンスの処理方針

- Gemini のレスポンスは JSON 文字列として受け取り、GAS 側で `JSON.parse()` する
- パースエラーの場合は要確認として処理し、OCR テキストのみ保存する
- Gemini が返す `needs_review: true` に加え、GAS 側バリデーションでも要確認チェックを行う（[5. 要確認判定ルール](#5-要確認判定ルール) 参照）

## 5. 要確認判定ルール

以下のいずれかに該当する場合、`needs_review` フラグを `TRUE` とし、`review_reason` に理由を記録する。

| # | 判定条件 | 要確認理由の例 |
|---|---------|--------------|
| 1 | 合計金額（税込）が取得できない / null | 「合計金額が取得できませんでした」 |
| 2 | 利用日または発行日が取得できない | 「利用日・発行日が取得できませんでした」 |
| 3 | 支払先名が取得できない | 「支払先名が取得できませんでした」 |
| 4 | 主税率区分が「不明」 | 「税率区分が判別できませんでした」 |
| 5 | 合計金額と税内訳の合計が一致しない（許容誤差 ±2 円以上） | 「合計金額と税内訳の合計が一致しません（合計: ○円、内訳合計: ○円）」 |
| 6 | 書類種別が「不明」 | 「書類種別が判定できませんでした」 |
| 7 | Gemini の `needs_review` が `true` | Gemini が返した `review_reason` をそのまま使用 |
| 8 | Gemini API 呼び出しがエラーまたはタイムアウト | 「Gemini API 呼び出しエラー: （エラー内容）」 |
| 9 | JSON パースエラー | 「Gemini レスポンスの JSON パースに失敗しました」 |

要確認理由が複数ある場合は改行区切りで連結して `review_reason` に保存する。

> **注意**：上記条件はあくまで自動判定の基準であり、フラグが立っていない場合でも OCR 精度の限界により誤りが含まれる可能性がある。申請者は必ず内容を確認してから経費申請に使用すること。

## 6. ファイル命名規則

処理結果に応じて、ファイル名の先頭に `yyyymmdd-{状態}-` 形式のプレフィックスを付与する。`yyyymmdd` は GAS がファイルを処理した日付を示す。

| 処理結果 | プレフィックス形式 | 例 |
|---------|-----------------|------|
| 正常処理済み | `yyyymmdd-処理済-` | `20260323-処理済-receipt_001.jpg` |
| 要確認 | `yyyymmdd-要確認-` | `20260323-要確認-IMG_2048.heic` |
| エラー | `yyyymmdd-エラー-` | `20260323-エラー-scan_003.png` |

### 6.1 処理済みファイル（要確認なし）

```
{yyyymmdd}-処理済-{元のファイル名}
```

**例：**
```
20260323-処理済-receipt_001.jpg
20260323-処理済-IMG_2048.heic
```

ファイルは `02_処理済み` フォルダへ移動する。

### 6.2 処理済みファイル（要確認あり）

```
{yyyymmdd}-要確認-{元のファイル名}
```

**例：**
```
20260323-要確認-receipt_002.jpg
20260323-要確認-IMG_2049.heic
```

ファイルは `03_要確認` フォルダへ移動する。ユーザーがスプレッドシートの内容を手動確認・修正した後、`02_処理済み` フォルダへ移動する。`02_処理済み` フォルダ内では `yyyymmdd-要確認-` プレフィックスが残るため、手作業で対応されたファイルであることが判別できる。

### 6.3 エラーファイル

```
{yyyymmdd}-エラー-{元のファイル名}
```

**例：**
```
20260323-エラー-receipt_003.jpg
20260323-エラー-scan_004.png
```

ファイルは `04_エラー` フォルダへ移動する。ユーザーがエラーを解消した後（手動でスプレッドシートに入力するなど）、`02_処理済み` フォルダへ移動する。`02_処理済み` フォルダ内では `yyyymmdd-エラー-` プレフィックスが残るため、手作業で対応されたファイルであることが判別できる。

### 6.4 命名規則の補足

- `yyyymmdd` は GAS がファイルを処理した日付（例：`20260323`）
- 元のファイル名はアップロード時のファイル名をそのまま使用する
- `02_処理済み` フォルダ内のプレフィックスで処理経緯を確認できる：
  - `yyyymmdd-処理済-`：自動処理で正常完了
  - `yyyymmdd-要確認-`：ユーザーが手動確認・修正して対応済み
  - `yyyymmdd-エラー-`：ユーザーがエラーを解消して対応済み

## 7. 設定値例

GAS スクリプトの `CONFIG` オブジェクト設定例を以下に示す。実際の ID・名前はユーザーが設定する。

```javascript
const CONFIG = {
  // --- Google Drive ---
  // 経費証憑画像のアップロード先フォルダ
  INTAKE_FOLDER_ID: 'ここに取込フォルダIDを貼り付け',

  // 処理済みファイルの移動先フォルダ
  PROCESSED_FOLDER_ID: 'ここに処理済みフォルダIDを貼り付け',

  // --- Google Sheets ---
  // 経費証憑台帳スプレッドシートの ID
  SPREADSHEET_ID: 'ここにスプレッドシートIDを貼り付け',

  // 経費データを書き込むシート名
  SHEET_NAME: '経費証憑台帳',

  // 処理ログを書き込むシート名
  LOG_SHEET_NAME: '処理ログ',

  // --- 精算対象月の決定方式 ---
  // 'fixed'  : EXPENSE_MONTH_FIXED の値を使用
  // 'folder' : 取込フォルダ名（YYYY-MM 形式）から自動取得
  EXPENSE_MONTH_MODE: 'fixed',

  // EXPENSE_MONTH_MODE = 'fixed' の場合の精算対象月（毎月更新が必要）
  EXPENSE_MONTH_FIXED: '2026-03',

  // --- 申請者情報 ---
  APPLICANT_NAME: 'ここに申請者名またはメールアドレスを設定',

  // --- 処理制御 ---
  // 1 回のトリガー実行で処理するファイルの最大件数（GAS 実行時間制限対応）
  MAX_FILES_PER_RUN: 10,

  // --- Gemini API ---
  // 使用するモデル名（GEMINI_API_KEY はスクリプトプロパティで管理）
  GEMINI_MODEL: 'gemini-2.5-flash-lite'
};
```

### 7.1 精算対象月決定方式の選択

#### `EXPENSE_MONTH_MODE: 'fixed'` の場合

- `EXPENSE_MONTH_FIXED` に当月の精算対象月を設定する
- 月次処理のタイミングで毎回この値を更新する運用が必要

#### `EXPENSE_MONTH_MODE: 'folder'` の場合

取込フォルダ内に月別サブフォルダを作成し、フォルダ名（`YYYY-MM`）から精算対象月を自動取得する。

```
01_経費証憑取込/
├── 2026-03/   ← このフォルダ配下のファイルは精算対象月 = 2026-03
└── 2026-04/   ← このフォルダ配下のファイルは精算対象月 = 2026-04
```

フォルダ名が `YYYY-MM` 形式でない場合は要確認フラグを付与し、精算対象月を空欄にする。

## 8. セキュリティ設計

### 8.1 API キー管理

| 項目 | 方針 |
|------|------|
| `GEMINI_API_KEY` の保存場所 | GAS の「プロジェクトの設定 > スクリプトプロパティ」に登録する |
| コードへの記載 | **絶対に記載しない**。コミット前に `git diff` で実キーが含まれていないことを確認する |
| 取得方法 | `PropertiesService.getScriptProperties().getProperty('GEMINI_API_KEY')` |
| 定期的なキーローテーション | 利用規約に従い、定期的なキーの再発行を推奨する（将来検討） |

### 8.2 アクセス権限管理

| リソース | アクセス制御方針 |
|---------|----------------|
| 取込フォルダ | 申請者のみ書き込み可能。管理者は編集可能 |
| 処理済みフォルダ | 申請者は閲覧のみ可能。管理者は編集可能 |
| スプレッドシート | 申請者は自分の行のみ編集可能（理想）または申請者・管理者のみアクセス可能 |
| GAS スクリプト | GAS スクリプトエディタへのアクセスは管理者のみ |

> **重要**：「リンクを知っている全員が閲覧可能」には設定しないこと。経費証憑には個人の購買情報が含まれる。

### 8.3 個人情報保護

- レシート・領収書には個人の購買行動・移動情報・クレジットカード情報が含まれる場合がある
- Gemini API に画像を送信する際は、Google の [利用規約](https://policies.google.com/terms) および [プライバシーポリシー](https://policies.google.com/privacy) に従う
- 画像データを Gemini API に送信することについて、社内の情報セキュリティポリシーに照らした確認を行うこと

### 8.4 GAS OAuth スコープ

GAS スクリプトには以下の最小限のスコープのみを付与する。

| スコープ | 用途 |
|---------|------|
| `https://www.googleapis.com/auth/drive` | Drive ファイルの読み取り・移動 |
| `https://www.googleapis.com/auth/spreadsheets` | スプレッドシートへの追記 |
| `https://www.googleapis.com/auth/script.external_request` | Gemini API への HTTP リクエスト |
